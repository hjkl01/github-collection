### [katanemo plano](https://github.com/katanemo/plano)  ![GitHub Repo stars](https://img.shields.io/github/stars/katanemo/plano?style=social)

**项目核心内容总结：**

**项目名称：** Plano  
**项目定位：** 一个面向智能代理（agentic）应用的 AI 原生代理服务器和数据平面，简化代理应用的开发和部署。

---

### **核心功能：**

1. **代理路由与编排（Orchestration）：**  
   提供低延迟的代理间路由能力，无需修改应用代码即可添加新代理。

2. **模型敏捷（Model Agility）：**  
   支持通过模型名、别名或偏好策略自动路由大模型请求，统一管理不同模型和提供商。

3. **代理信号捕获（Agentic Signals）：**  
   零代码自动捕获代理行为的信号和 OpenTelemetry 追踪，便于持续优化和评估。

4. **过滤链（Filter Chains）：**  
   提供统一接口实现安全防护、内容审核和记忆功能，增强代理应用的安全性与可控性。

---

### **使用方法：**

1. **定义代理配置（YAML）：**  
   通过 YAML 文件声明代理的 URL 和描述，Plano 自动处理路由逻辑、模型管理和追踪。

2. **编写简单代理代码：**  
   代理只需实现 OpenAI 兼容的 HTTP 服务接口，Plano 处理模型调用和路由逻辑。

3. **启动 Plano：**  
   使用 `planoai up config.yaml` 命令启动服务，即可开始测试代理交互。

4. **自动追踪与信号捕获：**  
   每个请求都会自动追踪并记录日志，无需额外编码即可获得完整可观测性。

---

### **主要特性：**

- 支持任意语言或 AI 框架；
- 提供开箱即用的代理编排、模型路由和可观测性；
- 通过轻量级模型（如 4B 参数的 Orchestrator）实现高效路由；
- 提供统一接口，简化代理应用的开发和部署流程；
- 与 Envoy 基础设施集成，适合大规模现代应用环境。

---

### **适用场景：**

适合需要构建、部署和管理多个智能代理（如聊天机器人、旅行助手、客服系统等）的应用场景，尤其适用于希望提升代理应用的可观察性、安全性与模型灵活性的开发者。
