### [MichaelMure git-bug](https://github.com/MichaelMure/git-bug)

git-bug 是一个基于 Git 的分布式、离线优先的问题跟踪工具，将问题、评论等作为 Git 对象嵌入仓库，支持推拉同步到多个远程仓库。  

**核心功能与特性**  
- **原生 Git 存储**：所有问题、用户信息、评论以 Git 对象形式存储，无需额外文件，便于版本管理。  
- **分布式与版本化**：利用 Git 的去中心化架构，支持离线操作及后续同步。  
- **高效性能**：毫秒级问题列表与搜索速度。  
- **第三方平台同步**：通过“桥梁”功能与 GitHub、GitLab 等平台对接。  
- **多接口支持**：提供 CLI、TUI 和 Web 浏览器多种交互方式。  
- **易集成**：无需复杂配置即可在仓库中启用问题管理。  

**使用方法**  
1. 安装：参考 [INSTALLATION.md](doc/INSTALLATION.md) 或下载最新版本。  
2. 学习：通过 [文档](doc) 掌握操作。  
3. 参与：贡献代码或在 [GitHub 讨论区](https://github.com/git-bug/git-bug/discussions) 交流。  
4. 社区支持：通过 Matrix（`#git-bug:matrix.org`）实时沟通。
