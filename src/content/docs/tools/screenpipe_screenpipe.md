### [screenpipe screenpipe](https://github.com/screenpipe/screenpipe)  ![GitHub Repo stars](https://img.shields.io/github/stars/screenpipe/screenpipe?style=social)

**项目名称**：screenpipe（屏幕管道）

**项目功能**：  
screenpipe 是一款开源应用，能持续记录你的屏幕和音频内容，建立一个可搜索、AI 驱动的记忆库，帮助你随时回溯在电脑上看到、听到或做过的内容。所有数据本地存储，隐私安全，支持多平台使用。

**使用方法**：  
- **CLI（命令行工具）**：  
  使用 `bunx screenpipe@latest record` 或 `npx screenpipe@latest record` 启动记录。  
  也可通过 shell 脚本安装并运行（支持 Mac/Linux/Windows）。  
- **桌面应用**：  
  下载桌面应用（[screenpi.pe](https://screenpi.pe)），一次性购买，包含所有功能和自动更新。

**主要特性**：  
1. **记录一切**：同时记录所有连接的屏幕和音频，支持 OCR 提取文本、语音转文字。  
2. **AI 搜索**：支持自然语言搜索，能按时间、应用、窗口标题、浏览器 URL 等条件筛选结果。  
3. **完全本地**：所有数据存储在本地，支持使用本地 AI 模型（如 Ollama），确保隐私安全。  
4. **插件系统（Pipes）**：可扩展架构，支持与 Obsidian、Notion、会议助手等集成，开发者可自定义插件。  
5. **MCP 协议支持**：与 Claude、Cursor 等 AI 工具集成，提供上下文查询功能。  
6. **时间轴视图**：可视化的记录时间线，可像电视回放一样浏览历史屏幕内容。  
7. **API 接口**：提供 REST API 和 SDK，支持开发者自定义功能。  
8. **跨平台支持**：支持 macOS（含 Apple Silicon）、Windows 10/11、Linux。  
9. **资源占用**：CPU 占用约 5%-15%，内存 0.5-3GB，月存储约 15GB。  

**适用人群**：  
- 需要查找历史屏幕内容的知识工作者  
- 想为 AI 编程助手提供上下文的开发者  
- 需要记录会议内容的远程工作者  
- 注意力不集中、常忘记文件或对话的人群  
- 需要本地隐私替代方案的用户  

**比较优势**：  
- 开源、MIT 许可证  
- 数据完全本地  
- 支持多显示器、语音转文字  
- 提供完整 API 和插件系统  
- 相比 Rewind、Microsoft Recall 等，更注重隐私和扩展性
